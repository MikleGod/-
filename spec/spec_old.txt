1.1	Окно звонков

1	Должно открываться после запуска приложения
2	Должно включать список наименований типа: «Дата начала звонка «ГГ:ММ:ДД,  ЧЧ:TT» (г – год, м – месяц, д – день, ч – час, т - минута); продолжительность звонка; ФИО клиента; контактный телефон клиента; результат звонка (возможные результаты: «Встреча», «Перезвонить», «Отказ»; При нажатии на поле «Встреча» должно открыться «Окно подробностей» для данной встречи)»
3	Кнопку с функциональностью добавления записи в список. Кнопка открывает «Окно подробностей» для добавления элемента списка
4	По нажатию на элемент из списка должно открыться окно с подробностями звонка «Окно подробностей»
5	Должна быть возможность перейти на «Окно встреч»

1.2	Окно встреч

1	Может быть впервые открыто только из «Окна звонков».
2	Должно включать список наименований для встреч. Содержание элемента зависит от типа встречи.
3	Встречи могут быть двух типов: «Запланированная» и «Завершенная».
4	Содержание «Запланированной»: «Дата начала «ГГ:ММ:ДД,  ЧЧ:TT» (г – год, м – месяц, д – день, ч – час, т - минута); ФИО клиента; телефон клиента; функциональность для переноса встречи (возможно, кнопка)».
5	Содержание «Завершенной»: «Дата начала «ГГ:ММ:ДД,  ЧЧ:TT» (г – год, м – месяц, д – день, ч – час, т - минута); ФИО клиента; адрес клиента: телефон клиента; продолжительность встречи; результат встречи (возможные варианты: «Встреча», «Отказ», «Звонок», «Договор»)».
6	Кнопку с функциональностью добавления записи в список. Кнопка открывает «Окно подробностей» для добавления элемента списка.
7	По нажатию на элемент из списка должно открыться окно с подробностями звонка «Окно подробностей».
8	Должна быть возможность перейти на «Окно звонков».

1.3	Окно подробностей
•	Это окно может использоваться в четырех целях: показать информацию о звонке или о встрече (запланированной или завершенной), изменить информацию о встрече (запланированной или завершенной) или звонке, добавить звонок или встречу, завершить встречу.
1.3.1	Информация о звонке
•	 
1.	Все пустые поля должны быть заполнены информацией из базы данных об этом звонке.
2.	Поле результат – выпадающий список. Может состоять из (см. п. 1.1.1).
3.	Если результат – «Встреча», то по нажатию на это поле должно открыться «Окно подробностей» для этой встречи.
4.	При нажатии на кнопку «Изменить» открывается окно «Добавление звонка».

1.3.2 Информация завершенной о встрече
1.	Дата
2.	Имя клиента
3.	Номер телефона клиента
4.	Адрес объекта
5.	Стоимость объекта
6.	Стоимость квадратного метра
7.	Площадь (Общая площадь/Жилая площадь/Площадь кухни).
8.	Описание встречи.
9.	 Дальнейшие действия (Набрать в мае, перезвонить завтра, встретиться во вторник, больше там никогда не появляться и т.п.)

1.3.3 Добавление звонка
•	 
1.	Пользователь должен заполнить все незаполненные поля, кроме поля «Краткое описание».
2.	Выбрать результат из выпадающего списка «см. п. 1.1»
3.	Если выбран результат «Встреча», то должно открыться окно «Добавление встречи».
4.	Сохранение происходит если пункт 1 выполнен и, если выбрана «Встреча», пользователь заполнил все верно в открывшемся окне и сохранил там данные, в противном случае поле должно остаться пустым.
5.	Кнопка «Сохранить» сохраняет результаты работы с окном и закрывает его. Если пункт 1 не выполнен, то показать сообщение об ошибке.
6.	Сообщение об ошибке должно содержать название поле, которое не прошло валидацию.
